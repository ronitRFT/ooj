# OOJ Event Management System

## Application Overview

### 1. Introduction

The **OOJ Event Management System** is a centralized web and mobile application designed to simplify the planning, execution, and management of events organized by the OOJ Foundation.

The current event management process involves multiple manual activities such as maintaining guest lists in spreadsheets, sending invitations individually, managing registrations through calls or messages, and recording attendance manually during the event. These processes are time-consuming, prone to human error, and difficult to monitor in real time.

The proposed system will digitize the complete event lifecycle by providing a single platform for event creation, guest management, invitation distribution, registration, attendee check-in, and reporting.

The application aims to improve operational efficiency, reduce manual effort, and provide organizers with complete visibility into every stage of an event.

---

# 2. Objective

The primary objective of the OOJ Event Management System is to provide a secure, efficient, and user-friendly platform that enables the foundation to manage events digitally from start to finish.

The system is intended to:

* Eliminate manual record keeping.
* Streamline invitation and registration processes.
* Enable digital attendance using QR codes.
* Provide real-time event monitoring.
* Generate reports for better decision-making.
* Improve the overall experience for both organizers and guests.

---

# 3. Target Users

The application is designed for four primary user groups.

## 3.1 Super Admin

The Super Admin manages the overall application and has complete control over system configuration, administrator accounts, and reporting.

Responsibilities include:

* Managing administrators
* Monitoring all events
* Viewing analytics and reports
* Configuring system settings

---

## 3.2 Admin

Admins are responsible for organizing and managing individual events.

Responsibilities include:

* Creating and managing events
* Managing guest information
* Sending invitations
* Monitoring registrations
* Viewing attendance
* Exporting reports

---

## 3.3 Volunteer

Volunteers assist during the event.

Responsibilities include:

* Scanning guest QR codes
* Checking in attendees
* Registering walk-in visitors (if permitted)

---

## 3.4 Guest

Guests interact with the system through invitations and registration pages.

Guests can:

* View event information
* Register for an event
* Receive a digital invitation
* Present their QR code for entry

---

# 4. Core Features

## Event Management

The application allows administrators to create and manage events by entering details such as:

* Event title
* Description
* Date and time
* Venue
* Organizer information
* Event banner
* Contact details
* Event status

---

## Guest Management

The system maintains a centralized guest database where administrators can:

* Add new guests
* Update guest information
* Search and filter guests
* Import guest lists
* Export guest data

---

## Invitation Management

The application generates personalized digital invitations for registered guests.

Each invitation contains:

* Guest information
* Event details
* Unique QR Code
* Venue information
* Date and time

Invitations can be shared digitally through supported communication channels.

---

## Registration Management

Guests can register through an online registration form.

The system automatically stores registration details and associates each registration with the corresponding event.

---

## QR-Based Check-In

Each registered guest receives a unique QR code.

On the day of the event, volunteers scan the QR code to:

* Verify guest identity
* Record attendance
* Prevent duplicate check-ins

This process enables faster and more accurate entry management.

---

## Dashboard

The dashboard provides organizers with a real-time overview of the event.

Typical information includes:

* Total guests
* Registered guests
* Checked-in attendees
* Pending invitations
* Event summary

---

## Reports

The application generates reports that help organizers evaluate event performance.

Reports may include:

* Registration reports
* Attendance reports
* Guest lists
* Invitation status
* Event summary

Reports can be exported for record keeping and future analysis.

---

# 5. Application Workflow

The overall workflow of the system is as follows:

1. An administrator creates a new event.
2. Guest information is added or imported.
3. Invitations are generated and shared.
4. Guests register through the registration portal.
5. Each registered guest receives a unique QR code.
6. On the event day, volunteers scan QR codes at the entrance.
7. Attendance is recorded in real time.
8. Organizers monitor progress through the dashboard.
9. Reports are generated after the event.
